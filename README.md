# Росреестр Парсер

Веб-приложение для загрузки и анализа данных из реестра аккредитованных лиц (РАЛ) `pub.fsa.gov.ru`.

## Что умеет

- Полный парсинг данных по всем регионам РФ
- Обновление базы только новыми записями
- Дашборд со статистикой и прогрессом загрузки
- Экспорт в Excel (`.xlsx`) с фильтрами по округу/региону
- Логи в реальном времени
- Настройка параметров парсинга и темы интерфейса

## Требования

- Python `3.12+`
- `uv` (менеджер зависимостей и запуска)

## Быстрый старт

```bash
uv sync
uv run streamlit run web_app.py
```

Откройте: `http://localhost:8501`

## Запуск для пользователя (Windows)

- Двойной клик по `run.bat`.
- Скрипт запустит UI и откроет страницу в браузере.

## Первый запуск

1. Откройте вкладку **Настройки**.
2. Вставьте `fgis_token` (инструкция есть прямо в UI).
3. Нажмите **Сохранить**.
4. Вернитесь на **Дашборд** и нажмите **Полный парсинг**.
   ![Копирование `fgis_token`](image.png)

## Кнопки на дашборде

- `Полный парсинг` — полная загрузка по всем регионам.
- `Закачать обновления` — загружает только новые записи.
- `Обновить данные источника` — обновляет эталонные totals по округам.
- `Скачать XLSX` — экспорт текущей выборки.

## Вкладка «Логи»

- Показывает актуальный хвост `logs/log.log`.
- Во время активного запуска обновляется автоматически.
- Позволяет остановить процесс кнопкой **Остановить**.

## CLI-режим (без UI)

```bash
uv run python main.py           # полный парсинг
uv run python main_update.py    # только новые записи
uv run python main_xlsx.py      # экспорт в XLSX
```

## Основные файлы проекта

```text
fsagov/
├── web_app.py         # Streamlit интерфейс
├── main.py            # полный парсинг
├── main_update.py     # обновление (новые записи)
├── main_xlsx.py       # экспорт в XLSX
├── src/utils/         # API, БД, XLSX, логирование
├── data/              # база и экспортные файлы
├── logs/              # логи
├── config.py          # базовые константы
└── config.json        # пользовательские настройки
```

## Полезно знать

- Парсинг может занимать продолжительное время (зависит от API и сети).
- Если вкладка браузера закрыта, фоновый процесс продолжает работать.
- Токен храните только локально и не публикуйте в логах/скриншотах.
